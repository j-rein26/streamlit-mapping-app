# Proximity-Based Geospatial Mapping Tool
## Tech Stack: Python, Google Sheets API, Geocoding Logic, Google Maps API

# 🗺️ Project Overview

I built this application to solve the logistical challenge of visiting multiple physical addresses efficiently. The app transforms a raw list of addresses into an optimized visual map, allowing users to plan routes based on physical proximity rather than a random list.

# 🚀 Key Features

**Automated Geocoding:** Converts physical addresses from a Google Sheet into precise latitude/longitude coordinates.

**Proximity Sorting:** Uses custom logic to order locations based on their distance to one another, minimizing travel time.

**Smart Filtering:** Users can toggle markers to see only the specific contacts they intend to visit, reducing visual clutter.

**Dynamic Data Sync:** Reads directly from a live Google Sheets document, ensuring the map is always up to date with the latest field data.

# 🛠 Technical Implementation

This project required handling API integration between Google Sheets and mapping services. It demonstrates my ability to take unstructured data and turn it into a functional, location-aware business tool.
